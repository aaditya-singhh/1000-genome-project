PURPOSE : Initial QC and exploratory analysis of Y chromosome VCF data.

--- TECHNIQUE: Delimiter Auto-Detection ---

--- TECHNIQUE: Quality Control (QC) Filtering ---
Parameter               Value        Why
QUAL threshold          > 30         Phred score >30 = <0.1% base-call error probability
FILTER column           == "PASS"    Only variants that passed all variant-caller checks
Biallelic filter        no comma in ALT  Multi-allelic sites removed for consistency
Call rate threshold     >= 0.95      Variants with <95% call rate are unreliable
Sample missingness      <= 0.10      Samples with >10% missing data removed

--- TECHNIQUE: Male Detection on Y Chromosome ---
Threshold : Call rate per sample > 0.05 (5%)

--- TECHNIQUE: PCA (300 Components) ---
n_components  : 300

Variance explained : 64.9%

--- TECHNIQUE: Data Augmentation (AugmentedDataset) ---
1. Gaussian Noise:
   noise_std : 0.05
   Applied to: Each feature during training (disabled at val/test)
   Why       : Prevents overfitting; simulates measurement noise

2. Random Masking:
   p_mask    : 0.10 (10% of features zeroed per sample)
   Why       : Simulates missing genotype data; improves generalisation

3. MixUp (batch-level):
   Distribution: Beta(alpha, alpha) lambda
   Formula   : x_mix = lambda*xi + (1-lambda)*xj
   Loss      : lambda*L(y_hat, ya) + (1-lambda)*L(y_hat, yb)
   Why       : Creates virtual training examples; smooths decision boundaries

--- TECHNIQUE: Focal Loss ---
Formula   : FL(p) = -alpha * (1-p)^gamma * log(p)
gamma     : 2.0  (standard; higher = more focus on hard examples)
weight    : Inverse class frequency (handles class imbalance)
Why       : AMR class (n=170) under-represented; focal loss emphasises hard samples

10. X_experiment1_ml.ipynb
PURPOSE : ML classification of 2504 X chromosome samples into 5 super-populations.
INNOVATION: Incremental Gram-Matrix PCA on all 3.4M variants using MPS GPU.

--- TECHNIQUE: Incremental Gram-Matrix PCA ---
Why: Full matrix (3.4M x 2504) = ~34 GB; standard PCA impossible.
Gram matrix G = X^T X is only 2504x2504 = ~50 MB.
G accumulated chunk by chunk; then eigendecomposed once.

Parameters:
  chunksize       : 50,000 variants per chunk
  N_COMPONENTS    : 50
  device          : Apple MPS (GPU)
  accumulation dtype : float64  (prevents precision loss over 3.4M additions)
  imputation      : row mean (per-variant mean across samples)

Gram Matrix Centering:
  Formula : G_centred = G_cpu - n_var * outer(mu, mu)
  Why     : PCA requires zero-mean data

Eigendecomposition:
  Library : numpy.linalg.eigh (symmetric matrix, faster and stable)

--- OUTPUT VALUES: Gram Matrix Computation ---
Variants processed : 3,449,249
Runtime            : 69.9 minutes (MPS GPU)
Eigendecomposition time : 1.1 seconds

--- OUTPUT VALUES: PCA ---
Variance explained by 50 PCs : 66.7%
PC1 : 55.2%
PC2 : 4.8%
